# Plan: Refactor CodeWriterHandler

## Goal
Refactor `CodeWriterHandler` in `/Users/derekw/Documents/projects/boiga/dendrophis/subagents/handlers/code_writer.py` to address the following issues:
1. **Centralized LLM Client**: Remove the isolated `LLMClient` instance and use the session's LLM client.
2. **Improved Error Handling**: Replace bare `except Exception` with specific exceptions and structured error messages.
3. **Type Safety**: Replace `Any` with specific types where possible.
4. **Logging**: Replace `print` statements with the `logging` module.
5. **Configuration**: Move hardcoded values (e.g., model name) to the config system.
6. **Dependency Injection**: Accept dependencies (e.g., tools, LLM client) via constructor for better testability.
7. **Code Organization**: Split large methods into smaller, focused helpers.

---

## Current State Analysis

### `CodeWriterHandler` Issues
1. **Isolated LLM Client** (lines 69-79):
   - Creates its own `LLMClient` with hardcoded model (`qwen/qwen3-coder:latest`).
   - Duplicates LLM client management logic already handled by the session.
   - Hardcodes config values (e.g., `temperature=0.1`, `top_k=64`).

2. **Error Handling** (lines 101-102, 134-140, 172-179, 200-219, 246-252):
   - Uses bare `except Exception` in multiple places.
   - Swallows errors silently or returns generic messages.

3. **Logging** (lines 143-151, 336-338, 367-368):
   - Uses `print` for debugging instead of `logging`.

4. **Type Safety**:
   - Overuses `Any` in method signatures and return types.

5. **Dependency Management**:
   - Tools (`ReadTool`, `EditTool`, `WriteTool`, `BashTool`) are instantiated internally.
   - No dependency injection for testability.

6. **Code Organization**:
   - `execute` method (lines 81-244) is ~160 lines long and handles multiple responsibilities.
   - `_parse_edits` (lines 300-338) and `_run_ruff` (lines 340-368) are tightly coupled to the handler.

## Proposed Changes

### 1. Dependency Injection
- **Add Constructor Parameters**:
  ```python
  def __init__(
      self,
      llm_client: LLMClient | None = None,
      read_tool: ReadTool | None = None,
      edit_tool: EditTool | None = None,
      write_tool: WriteTool | None = None,
      bash_tool: BashTool | None = None,
      config: DendrophisConfig | None = None,
  ) -> None:
  ```
- **Fallback to Defaults**: If dependencies are not provided, create them lazily (for backward compatibility).

### 2. Centralized LLM Client
- **Remove Hardcoded LLM Client**: Delete lines 69-79.
- **Use Injected LLM Client**: Use the `llm_client` parameter or fetch it from the session.
- **Configuration**: Use `config.llm` for model settings (e.g., `temperature`, `top_k`).

### 3. Error Handling
- **Define Custom Exceptions**:
  ```python
  class CodeWriterError(Exception):
      """Base exception for code-writer errors."""

  class FileReadError(CodeWriterError):
      """Failed to read a file."""

  class FileEditError(CodeWriterError):
      """Failed to edit a file."""

  class LLMCallError(CodeWriterError):
      """Failed to call the LLM."""

  class ParseError(CodeWriterError):
      """Failed to parse LLM response."""
  ```
- **Replace Bare `except Exception`**: Use specific exceptions and re-raise with context.

### 4. Logging
- **Add Logger**:
  ```python
  logger = logging.getLogger(__name__)
  ```
- **Replace `print`**: Use `logger.debug`, `logger.info`, `logger.warning`, `logger.error`.

### 5. Type Safety
- **Replace `Any` with Specific Types**:
  - Use `dict[str, str]` for `file_contents` and `python_analysis`.
  - Use `list[ToolCall]` for `tool_calls`.
  - Use `Path` for file paths.

### 6. Code Organization
- **Split `execute` Method**:
  - Extract file reading logic into `_read_files`.
  - Extract file analysis logic into `_analyze_files`.
  - Extract LLM prompt building into `_build_llm_prompt`.
  - Extract edit parsing into `_parse_edits` (already exists).
  - Extract edit execution into `_apply_edits`.
  - Extract Ruff execution into `_run_ruff` (already exists).

- **New Helper Methods**:
  ```python
  def _read_files(self, files: list[str]) -> tuple[dict[str, str], list[str]]:
      """Read file contents. Returns (file_contents, blockers)."""

  def _analyze_files(self, python_files: list[str]) -> tuple[dict[str, str], list[str]]:
      """Analyze Python files. Returns (python_analysis, blockers)."""

  def _build_llm_prompt(
      self, task: str, file_contents: dict[str, str], python_analysis: dict[str, str], context: dict[str, Any]
  ) -> str:
      """Build the LLM prompt."""

  def _apply_edits(self, edits: list[dict[str, str]]) -> tuple[list[dict[str, Any]], list[str]]:
      """Apply edits to files. Returns (changes, blockers)."""
  ```

### 7. Configuration
- **Add `code_writer_model` to `LLMConfig`**:
  ```python
  code_writer_model: str | None = None  # Override model for code-writer subagent
  ```
- **Use Config for Defaults**: Replace hardcoded values (e.g., `temperature=0.1`) with `config.llm.temperature`.
